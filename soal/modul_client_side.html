Archery Pop Game Documentation

  • MODULE_CLIENT_SIDE.doc - Documentation file

Introduction

In recent years internet has become our basic daily source and needs, enabling the dissemination of information in an inexhaustible content and interaction. Games have gained a prominent role in nowadays computer usage, allowing people to get access to entertainment from any places.

You are asked to develop a game called Archery Pop using HTML and CSS and develop client-side programming using JavaScript and its open source libraries. Some media files are available to you in a zip file. You can create more media and modify anything in the media if you want.

Resolution Requirement: Your game needs to be developed in a tablet resolution (960x600 pixels). In bigger resolution, the game must be centred in the screen both horizontally and vertically.

Description of Projects and Tasks

This is a module of 4 hours. Your first 2 hours must be used to create the design of the game in three PNG images and the initial layout using HTML/CSS. Your layout should follow the design that you created. The final 2 hours you will create the functionality of game using JavaScript that allows the game to work correctly in different web browsers.

Archery Pop Game Elements

  • Game Title
  • Board area 960 x 600 pixels
  • Custom cursor (scope/crosshair image)
  • Falling targets (archery board images)
  • Player score display
  • Total time / countdown timer
  • Lives/hearts indicator

Design and Initial Layout

Deliver at least 3 PNG image files that present:

  1. Game instruction: The first screen of the game presents the instructions to the player, a text field for player's name, and the Play button for starting the game. The "Play" button should be disabled if the text field is empty.
  2. Game board: It must present all elements described above in the game screen including score, timer, lives, and the playing area.
  3. Game icon: which represents the game immediately.
Important: Develop the initial markup (HTML + CSS) of your game application. Overall screen must be within 1024x1024 pixels.

Buttons must have active and hover effects. The buttons background in hover state must be #5d96d4 and the active state must use #0069d9.

The HTML and CSS code must be valid in the W3C standards for HTML5 and CSS3 rules in accordance with the WCAG and standard ARIA (Accessible Rich Internet Applications Suite).

Game Functionalities

Menu & Setup

  • Show game icon on the top of the screen.
  • Show game instruction in the centre after page is loaded.
  • Show text field for player to input their name.
  • Player can start game after clicking "Play" button at the bottom of game instruction.
  • The "Play" button should be disabled if the text field is empty.

Cursor & Aiming

  • Default cursor must be hidden inside the game board.
  • Custom cursor image (scope/crosshair) follows mouse movement inside the game board.
  • Scope image must be centered on mouse position.

Targets

  • Targets (archery board images) spawn at random X positions at the top of the board.
  • Targets fall from top to bottom at constant speed (3 pixels per frame).
  • Target spawn rate: 1 target every 2 seconds.
  • Maximum 5 targets on screen at the same time.
  • Minimum 1 target must always be present on screen.

Scoring System

Action Score Change
Initial score 0
Click on target (hit) +10 points, target disappears immediately
Target reaches bottom without being hit -5 points, lose 1 life

Lives System

  • Player starts with 3 lives (displayed as heart icons)
  • Lose 1 life when target reaches bottom
  • Game over when lives reach 0

Timer

  • Countdown timer starts at 60 seconds
  • Timer displayed at top of board in MM:SS format
  • Game ends when timer reaches 00:00

Game Over

Game ends when timer reaches 0 OR lives reach 0. Display game over screen with:

  • Final score
  • High score (saved in localStorage)
  • "New High Score!" message if applicable
  • "Play Again" button to restart
  • "Main Menu" button to return to instructions

Pause Feature

  • Player can pause game by pressing "Space" or clicking "Pause" button
  • Pause screen overlay with "Resume" and "Quit" buttons
  • All animations and timers stop during pause

Additional Requirements

  • Your game should work without JavaScript errors or messages shown in the browser console.
  • Maintain your HTML/CSS and JavaScript code organized and clean to facilitate future maintenance.
  • Use correct indentation and comments.
  • Use meaningful variable names and document your code as much as possible so another developer would be able to modify your work in the future.
  • The game needs to work correctly in Google Chrome.

Instructions for Competitors

The media files are available in the ZIP file. You can modify the supplied files and create new media files to ensure the correct functionality and improve the application.

Required Media Files

  • scope.png - Crosshair/scope cursor image (approximately 50x50 pixels)
  • target.png - Archery board target image (approximately 80x80 pixels)
  • heart.png - Life indicator icon
